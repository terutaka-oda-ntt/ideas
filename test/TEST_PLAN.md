# テスト計画

## 概要
このドキュメントは、`ideas`リポジトリのテスト戦略・計画を記述しています。

リポジトリの構成：
- ドキュメンテーションリポジトリ（Markdownファイル中心）
- CI/CDおよび開発環境に関する情報を含む
- pre-commitによる秘密検出フレームワークを導入

## テスト目的

**目標**: コミット前に以下について自動検証し、ドキュメント品質を保証する
- Markdownファイルの形式と内容の妥当性
- YAML形式の正確性
- ファイル構造の整合性
- 秘密情報の非存在

## テストケース

### テストカテゴリ1: ファイル形式チェック

| TC# | テスト項目 | 対象ファイル | 検証内容 | 期待結果 |
|:---|:---|:---|:---|:---|
| TC-01 | Markdown形式チェック | *.md | ファイルが有効なMarkdown形式であることを確認 | ✓ Parse成功 |
| TC-02 | YAML形式チェック | .pre-commit-config.yaml | YAML形式が正確で、必須フィールドが存在することを確認 | ✓ Parse成功 |
| TC-03 | ファイル末尾確認 | すべてのテキストファイル | ファイルが改行で終了していることを確認 | ✓ すべて改行で終了 |
| TC-04 | 末尾空白チェック | すべてのテキストファイル | 行末に不要な空白がないことを確認 | ✓ 空白なし |

### テストカテゴリ2: 内容チェック

| TC# | テスト項目 | 対象ファイル | 検証内容 | 期待結果 |
|:---|:---|:---|:---|:---|

### テストカテゴリ3: セキュリティチェック

| TC# | テスト項目 | 対象ファイル | 検証内容 | 期待結果 |
|:---|:---|:---|:---|:---|
| TC-08 | 秘密鍵検出 | すべてのファイル | 秘密鍵が含まれていないことを確認 | ✓ 検出なし |

### テストカテゴリ4: pre-commitフック動作確認

**重要**: これらのテストは、実際のgitフックが正常に動作するかを確認します。独立した一時的なgitリポジトリで実行されるため、本リポジトリに影響を与えません。

| TC# | テスト項目 | テスト方法 | 検証内容 | 期待結果 |
|:---|:---|:---|:---|:---|
| TC-11 | 秘密鍵ファイル検出 | 秘密鍵を含むファイルでコミット試行 | pre-commitが秘密鍵を検出し、コミットを中断 | ✗ コミット中止 |
| TC-12 | APIキー検出 | APIキーを含むファイルでコミット試行 | pre-commitがAPIキーを検出し、コミットを中断 | ✗ コミット中止 |
| TC-13 | 末尾空白の自動修正 | 末尾に空白があるファイルでコミット試行 | trailing-whitespaceフックが自動修正 | ✓ ファイル修正 |
| TC-14 | ファイル末尾の自動修正 | 改行を含まないファイルでコミット試行 | end-of-file-fixerフックが自動修正 | ✓ ファイル修正 |
| TC-15 | 正常なファイルのコミット成功 | 秘密情報を含まない正常なファイルでコミット | コミットが成功 | ✓ コミット成功 |
| TC-16 | YAML形式エラー検出 | 不正なYAML形式のファイルでコミット試行 | check-yamlフックがエラーを検出 | ✗ コミット中止 |
| TC-17 | 大容量ファイル検出 | 1MBを超えるファイルでコミット試行 | check-added-large-filesが検出 | ✗ コミット中止 |
| TC-18 | DB接続文字列検出 | 認証情報を含むDB接続文字列でコミット試行 | detect-secretsが検出し、コミットを中断 | ✗ コミット中止 |
| TC-19 | ログイン情報検出 | username/passwordを含むファイルでコミット試行 | detect-secretsが検出し、コミットを中断 | ✗ コミット中止 |

## テスト実行方法

### 全テスト実行
```bash
cd /workspaces/ideas
./test/run_tests.sh
```

### 個別テスト実行
```bash
# ファイル構造チェック
python3 test/test_structure.py

# YAML形式チェック
python3 test/test_yaml.py

# Markdownチェック
python3 test/test_markdown.py

# セキュリティチェック（リポジトリの静的解析）
python3 test/test_security.py

# pre-commitフック動作確認（TC-11～TC-19）
python3 test/test_precommit_hooks.py
```

## テスト実施スケジュール

- **ローカル開発時**: `pre-commit run --all-files` で全チェック実行
- **コミット前**: 自動的にpre-commitフックが実行
- **PR作成時**: GitHub Actionsで全テスト実行（予定）

## 合格基準

- 全テストケースが成功（Pass）
- 秘密情報が検出されない
- ファイル形式エラーが0件

## テスト環境

- Python: 3.x
- Operating System: Linux（Ubuntu 24.04）
- 必要なツール:
  - pyyaml（YAML解析）
  - markdown（Markdown解析）
  - detect-secrets（秘密検出）
